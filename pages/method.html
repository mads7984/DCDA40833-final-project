Research Methodology

How I collected, organized, analyzed, and presented the data for this project

Introduction

My methodology changed as the project developed. In the original proposal, I planned to rely much more heavily on large-scale web scraping across several coupon platforms. Once I started working with the actual sites, it became clear that this approach was too broad and too dependent on inconsistent page structures.

Because of that, I narrowed the scope to selected Type 2 diabetes medications and shifted to a more controlled comparison. This allowed me to build a cleaner dataset and focus on the variables most relevant to the final research question: listed price, availability, and transparency.

Final Research Question

The final version of the project asks how TrumpRx compares to GoodRx, SingleCare, and Cost Plus Drugs for selected Type 2 diabetes medications in terms of listed price, drug availability, and pricing transparency.

Narrowing the project in this way was necessary because it shifted the focus from trying to compare entire platforms at a very large scale to building a clearer and more usable comparison of a smaller set of drugs.

Data Collection

I began with TrumpRx and used it as the starting point for the project. From there, I selected a smaller group of Type 2 diabetes medications that could be compared more consistently across platforms. The final set included Ozempic, Rybelsus, Jardiance, Farxiga, Synjardy, and Xigduo XR.

For each drug, I collected data from TrumpRx, GoodRx, SingleCare, and Cost Plus Drugs. The variables I recorded included whether the drug was listed, the listed price, the price type, dosage, quantity, number of clicks required to access the price, a transparency score, and a direct link to the platform page when available.

Data Management

I organized the information into a structured CSV file with one row for each drug-platform combination. This made it easier to keep the dataset consistent and to use the same data across Python, Tableau, and the website.

When a drug was not listed on a platform, I still kept that row in the dataset and marked it as not available. That choice was important because non-availability is part of the comparison. Removing those rows would have hidden one of the key differences between the platforms.

Transparency Framework

One of the most important methodological decisions I made was to include a transparency score. I did this because I realized that affordability alone was not enough to evaluate the platforms. Some websites made prices easy to find and understand, while others required more steps or left important details unclear.

The transparency score was based on whether the site clearly displayed price, dosage, quantity, price type, restrictions, pharmacy variation, and ease of access. Each category was scored as either present or not present, and the total became the transparency score for that row.

Data Analysis

I used Python to clean the dataset, recalculate transparency scores, and generate the outputs used in the project. The analysis included availability by site, average price by site, median price by site, the cheapest site per drug, and price differences from TrumpRx where direct comparisons were possible.

I also used Python to generate chart images and output files that could be used in Tableau and on the website. This made the project easier to manage because once the dataset was structured correctly, I could create multiple outputs from the same source file.

AI Tool Usage

I used AI tools throughout the project, especially when troubleshooting scraping attempts, revising Python code, organizing HTML pages, and refining the structure of the website. AI was most useful when I needed help debugging, clarifying code logic, and narrowing the project to a more realistic scope.

At the same time, one of the biggest lessons from this project was that AI-generated code still has to be reviewed line by line. Some solutions looked correct at first but did not actually work with my file structure or dataset. Using AI made some parts of the process faster, but it did not replace careful revision or critical thinking.

Methodological Reflection

Overall, my methods were primarily quantitative because the project depends on price comparisons, availability counts, and transparency scoring. At the same time, the project also includes a qualitative element because I had to interpret how sites communicated information and how user-friendly those experiences were.

Narrowing the scope ended up making the project stronger. Rather than trying to force a broader and less stable method, I was able to build a more focused comparison that is easier to defend and more useful to the reader.
